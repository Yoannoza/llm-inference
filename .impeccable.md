# infer-metrics — Design Context

## Users
Auteur (Yoannoza) + communauté open-source d'ingénieurs ML / SRE / curieux de l'inférence LLM. Le projet est en cours d'ouverture publique. Le dashboard est consulté soit en session de bench (devant un terminal), soit converti en captures partagées sur les réseaux et dans le README du repo.

## Brand Personality
Trois mots : **précis, artisanal, opinionné.**
Le ton n'est pas SaaS clinique ni cyberpunk neon — c'est un *instrument de mesure de laboratoire* : on regarde un cadran, pas un dashboard analytics. L'autorité vient de la sobriété et de la densité d'information, pas du chrome.

## Aesthetic Direction
- **Thème** : sombre, marron profond chaud (le user a explicitement demandé brun sombre).
- **Référence mentale** : un oscilloscope analogique posé sur un bureau en cuir, un cahier de notes d'expérience en tabac/sépia, un boîtier Braun d'époque, une page éditoriale du Whole Earth Catalog.
- **Anti-références** : dashboard SaaS générique (Stripe-like cards bleues), neon cyberpunk, "AI palette" cyan-purple sur noir, glassmorphism.

## Design Principles
1. **Chaque chiffre est un cadran.** Les métriques-mères sont énormes, en mono, avec des unités discrètes — comme la valeur affichée sur un instrument.
2. **Le marron n'est pas un thème dark relooké** — c'est la couleur dominante, tintée OKLCH chaude, neutres tirés vers le hue brun, accent cuivre/ambre rare (≤10%).
3. **Typo éditoriale + technique.** Sérif display (Gloock) pour le titre, sans neutre (Geist) pour le UI, mono (JetBrains Mono) pour TOUS les nombres avec `tabular-nums`.
4. **Pas de cartes uniformes.** La carte hero (TTFT/TPOT) est plus large que les autres ; débit/coût/VRAM sont alignées en bas. Hiérarchie par asymétrie.
5. **La courbe latence/débit est dessinée à la main en SVG** (pas Chart.js) avec des tick marks d'instrument et des annotations textuelles, pour que ça ressemble à un relevé, pas à un graphique de slide.
